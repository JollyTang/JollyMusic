# ListenMusic 需求追踪

## v1.0.0 - 基础版本 (已完成)

- [x] 后端项目初始化 (Node.js + Express + TypeScript)
- [x] B站 WBI 签名模块
- [x] B站视频信息获取 API (BV号 → 标题/封面/分P)
- [x] B站 DASH 音频流获取 API
- [x] 音频流后端代理 (绕过B站防盗链，支持 Range 请求)
- [x] 图片后端代理 (绕过B站封面防盗链)
- [x] SQLite 歌单数据库 (playlists/tracks/playlist_tracks)
- [x] 歌单 CRUD API
- [x] 前端 uni-app 项目初始化 (Vue 3 + TypeScript + Pinia)
- [x] 搜索页 (BV号输入、视频信息展示、分P选择、搜索历史)
- [x] 播放器页面 (封面旋转、进度条、播放/暂停、上下曲、播放模式)
- [x] 底部迷你播放栏
- [x] 歌单管理页面 (创建/删除歌单、添加/移除曲目)
- [x] 播放器和迷你播放栏的收藏按钮

## v1.1.0 - 本地化 + 分享码 (已完成)

### 架构改造

- [x] 歌单存储从后端 SQLite 迁移到客户端本地存储 (uni.setStorage)
- [x] 移除后端数据库和歌单 API 路由，后端变为纯代理服务 (无状态)
- [x] 前端 api.ts 移除歌单相关服务端调用
- [x] 前端 playlist store 重写为本地存储方案

### 分享码功能

- [x] 实现分享码生成 (歌单数据 → Base64 编码，精简模式仅存 BV号+cid)
- [x] 实现分享码导入 (Base64 解码 → 逐个请求B站获取曲目信息)
- [x] 歌单详情页添加「分享歌单」按钮 (生成分享码 + 一键复制)
- [x] 歌单列表页添加「导入分享码」入口 (粘贴分享码 + 进度展示)

### UI/体验优化

- [x] 浅色主题统一 (修复歌单页暗色残留)
- [x] tabBar 图标修复 (从1x1像素升级为81x81像素)
- [x] 导入分享码时的加载进度展示
